# 混淆矩阵代表性数据集选择说明

更新时间：2026-06-16

混淆矩阵用于观察模型具体错在什么类别上。由于本项目完整实验包含多个模型、多个数据集和多台设备，如果把所有混淆矩阵全部放入正文，图表数量会过多，反而削弱报告重点。因此正文只选择少量代表性数据集，完整结果仍保留在 `results/raw/` 中。

## 选择原则

代表性混淆矩阵数据集需要覆盖以下差异：

1. 二分类与多分类任务。
2. 小样本与大样本任务。
3. 低维特征与高维特征任务。
4. 同一数据家族中不同特征表达方式带来的错误模式变化。
5. 与本项目样本数扩展性和特征数扩展性两条主线保持一致。

## 选定数据集

### 1. `mfeat-morphological_2000rows_6feat_multiclass`

选择理由：

- 这是 10 分类手写数字识别任务，能展示多分类场景下模型容易混淆哪些类别。
- 样本数约 2000，特征数只有 6，代表低维多分类任务。
- 它属于 mfeat 系列，方便和高维 mfeat 数据集对比。

报告用途：

- 观察低维特征下，多分类模型是否能区分数字类别。
- 判断错误是否集中在少数容易混淆的类别上。

### 2. `mfeat-pixel_2000rows_240feat_multiclass`

选择理由：

- 与 `mfeat-morphological` 来自同一手写数字数据家族。
- 样本数同样约 2000，但特征数增加到 240，代表高维多分类任务。
- 这种“同样任务、不同特征表达”的对比非常适合分析特征数增加后的错误模式变化。

报告用途：

- 对比低维特征和高维特征下，模型混淆类别是否发生变化。
- 支撑特征数扩展性分析。

### 3. `pc1_1109rows_21feat_binclass`

选择理由：

- 这是小样本二分类任务，样本数约 1100，特征数约 21。
- 它来自样本数扩展性中的 A 组，代表小数据场景。
- 二分类混淆矩阵更容易解释模型是否偏向某一类。

报告用途：

- 展示小样本二分类中模型是否偏向多数类。
- 配合 `balanced_accuracy` 和 `macro_f1` 解释 accuracy 是否可能被类别不均衡误导。

### 4. `default_of_credit_card_clients_30000rows_23feat_binclass`

选择理由：

- 这是当前选中数据集中最大的数据集之一，样本数约 30000。
- 它是二分类任务，特征数约 23，与小样本 `pc1` 形成样本规模对比。
- 该任务具有现实金融风险预测背景，适合报告中解释实际应用意义。

报告用途：

- 展示大样本二分类中模型是否仍存在类别偏向。
- 支撑样本数扩展性分析。
- 与 `pc1` 对比，观察样本规模变化后错误模式是否改善。

## 为什么不全部展示

完整实验包含：

```text
12 个数据集 * 4 个模型 * 多台设备
```

如果所有混淆矩阵都放入正文，图表数量会过多，读者难以抓住主要结论。本文采用“代表性正文图 + 完整结果文件保留”的方式：

- 正文展示 4 个代表性数据集的混淆矩阵。
- 所有原始混淆矩阵继续保留在 `results/raw/`。
- 如有需要，可在附录或结果文件夹中查看完整矩阵。

这种做法既保证了报告可读性，也保留了实验可复查性。
